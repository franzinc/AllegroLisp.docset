*nan-double*

This constant has the value of IEEE double-float NaN (Not-a-Number). It is a legal floating-point value. In arithmetic operations, it produces itself in all cases. See the linked discussion for more details.

NaNs like this one can result from normal arithmetic operations, such zero divided by zero in a function with types declared and compiled so comp:trust-declarations-switch is true, but that same call run interpreted may instead signal an error.

See Floating-point infinities and NaNs, and floating-point underflow and overflow in implementation.htm. Also see *negative-infinity-single*, *infinity-double*, *infinity-single*, *negative-infinity-double*, and *nan-single*
